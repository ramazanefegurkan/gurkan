# Knowledge Register

<!-- Append-only. Non-obvious rules, recurring gotchas, and useful patterns.
     Only add entries that would save future agents from repeating investigation. -->

| # | When | Category | Learning |
|---|------|----------|----------|
| K001 | M001/S01/T01 | tooling | .NET 10 `dotnet new sln` creates `.slnx` format (XML-based), not the legacy `.sln`. Use `dotnet sln GurkanApi.slnx add ...` for project manipulation. |
| K002 | M001/S01/T01 | tooling | .NET 10 web API template no longer includes Swashbuckle. `AddOpenApi()` + `MapOpenApi()` only serves raw JSON spec. Need explicit `Swashbuckle.AspNetCore` package for Swagger UI. |
| K003 | M001/S01/T01 | tooling | `Microsoft.AspNetCore.Identity` package is deprecated/empty on .NET 10. `PasswordHasher<T>` lives in the ASP.NET Core shared framework (`Microsoft.Extensions.Identity.Core`) — no extra package needed. |
| K004 | M001/S01/T01 | infra | Host port 5432 is occupied by `infra-postgres-1`. Gurkan project uses port 5434 for PostgreSQL. Connection strings must use `Port=5434`. |
| K005 | M001/S01/T01 | ef-core | EF Core Design package 9.x CLI works fine against 10.x runtime with a version mismatch warning. Migration generation and database update both succeed. |
| K006 | M001/S01/T04 | testing | WebApplicationFactory test deserialization must use `JsonStringEnumConverter` because Program.cs configures enum-as-string JSON serialization. Default `GetFromJsonAsync<T>` fails with "JSON value could not be converted to enum" on any response containing enum properties. Use a shared `ApiJsonOptions` instance. |
| K007 | M001/S01/T04 | testing | When integration tests share a single PostgreSQL test DB via `IClassFixture<WebApplicationFactory>`, xUnit's default parallel execution causes TRUNCATE races between test classes. Disable with `[assembly: CollectionBehavior(DisableTestParallelization = true)]`. |
| K008 | M001/S01/T04 | testing | `EnsureCreated()` and `MigrateAsync()` are incompatible on the same DB — EnsureCreated doesn't create `__EFMigrationsHistory`, so subsequent MigrateAsync tries to re-create existing tables and fails with 42P07. Use one approach consistently. For integration tests with Program.cs that calls MigrateAsync, pre-drop the DB and let migrations run. |
| K009 | M001/S02/T03 | frontend | Vite + React-TS template (2025+) sets `erasableSyntaxOnly: true` in tsconfig.app.json. This forbids TypeScript `enum` declarations (they emit runtime code). Use `const` objects with `as const` + derived union types instead: `export const Foo = { A: 0, B: 1 } as const; export type Foo = (typeof Foo)[keyof typeof Foo];` |
| K010 | M001/S02/T03 | backend | Backend JWT uses full XML namespace claim keys: `ClaimTypes.NameIdentifier` → `http://schemas.xmlsoap.org/ws/2005/05/identity/claims/nameidentifier`, `ClaimTypes.Email` → `http://schemas.xmlsoap.org/ws/2005/05/identity/claims/emailaddress`, `ClaimTypes.Role` → `http://schemas.microsoft.com/ws/2008/06/identity/claims/role`. No `FullName` claim exists in the token. |
| K011 | M001/S02/T04 | frontend | Backend `JsonStringEnumConverter` serializes C# enums as strings ("Apartment", "TRY"), not integers (0, 1). Frontend TypeScript const objects must use string values matching the API output, not numeric C# enum ordinals. Numeric values silently fail to match any label in lookup tables. |
| K012 | M001/S03/T04 | frontend | PostgreSQL with Npgsql rejects `DateTime.Kind = Unspecified` — happens when JSON deserializes date-only strings like "2025-01-01". All frontend date-picker values must be sent as UTC ISO strings by appending `T00:00:00Z` (e.g., "2025-01-01T00:00:00Z"). Use a `toUtcIso()` helper on all form date fields. |
| K013 | M001/S03/T04 | frontend | React Router nested routes with `<Outlet>` are the right pattern for persistent tab navigation across property sub-pages. PropertyLayout provides back-link + title + tabs, child routes render inside `<Outlet>`. Avoids duplicating navigation in every child component. |
| K014 | M001/S04/T03 | infra | Backend API serves on `localhost:5039` per `launchSettings.json`, not `localhost:5000`. The `api/client.ts` baseURL must use port 5039. This was a latent mismatch from S02/T03 that went unnoticed until browser-verified CRUD was attempted. |
| K015 | M001/S06/T02 | library | QuestPDF `Settings.License` setter triggers native Skia DLL loading via static initializer chain. In AnyCPU test hosts (xUnit + WebApplicationFactory), the native `QuestPdfSkia.dll` fails to load with `DllNotFoundException`. Wrap the license assignment in `try-catch (TypeInitializationException)` so the app boots in test environments. PDF export still works in production where `dotnet run` picks the correct RID. |
| K016 | M001/S06/T02 | backend | `QuestPDF.Fluent.Document` conflicts with `GurkanApi.Entities.Document` — use fully-qualified `QuestPDF.Fluent.Document.Create()` in controllers that import both namespaces. |
| K017 | M003/S01/T02 | infra | .NET 10 Debian-based `aspnet:10.0` image produces ~486MB final images. Alpine variants exist (`aspnet:10.0-alpine`) but may lack native Skia libraries needed by QuestPDF (K015). Don't switch to Alpine for projects using QuestPDF without testing PDF generation. The `nginx:alpine` base is ~57MB — frontend images around 90MB are normal. |
| K018 | M003/S01/T02 | infra | .NET 10 container images default to port 8080 (since .NET 8+) and run as non-root user with UID 1654 (`$APP_UID`). Use `chown $APP_UID:$APP_UID` for writable directories like uploads. Don't try to bind port 5000 — it won't work without overriding `ASPNETCORE_URLS`. |
| K019 | M003/S01/T03 | infra | Caddy treats `localhost` as a domain name and auto-provisions a self-signed TLS certificate via its internal CA, then redirects HTTP→HTTPS. For plain HTTP local testing, use `:80` as the site address (bare port), not `localhost`. In the Caddyfile, use `{$SITE_ADDRESS::80}` with a separate env var — DOMAIN stays for CORS/app config, SITE_ADDRESS controls Caddy's listening behavior. |
| K020 | M003/S01/T04 | bash | `((var++))` returns exit code 1 when `var` is 0, because the pre-increment expression evaluates to 0 (falsy in bash arithmetic). Combined with `set -e`, this silently kills the script. Use `var=$((var + 1))` for safe increment. Similarly, avoid `set -e` in test-runner scripts where `eval`'d commands are expected to return non-zero. |
| K021 | M003/S02/T01 | frontend | `window.addEventListener('storage', ...)` only fires in *other* tabs/windows — NOT in the same page that called `localStorage.setItem()`. For same-tab sync between non-React code (e.g., Axios interceptor) and React context, use a callback registration pattern instead. |
| K022 | M003/S03/T02 | ef-core | EF Core cannot translate `Dictionary.Values.SelectMany(t => t).Select(t => t.Id)` inside a LINQ `Where()` predicate — the `SelectMany(t => t)` on in-memory Dictionary values isn't SQL-translatable. Materialize the IDs into a `List<Guid>` first, then use `.Contains()` with that local list. |
| K023 | M003/S03/T02 | testing | Tenant creation endpoint auto-generates RentPayment records for each month in the lease period. Integration tests for rent payment import must account for these pre-existing records — use before/after count patterns or filter by `Notes == "Imported from CSV"` rather than asserting exact counts. |
| K024 | M003/S04/T01 | expo | Expo SDK 54 default template (create-expo-app --template default) ships with `expo-secure-store` and `expo-font` already in the `app.json` plugins array. No need to manually add them. The template also includes `expo-router`, `@expo/vector-icons`, `react-native-reanimated`, `expo-splash-screen`, and `react-native-gesture-handler` pre-installed. |
| K025 | M003/S04/T01 | expo | Expo Router `Stack.Protected` pattern requires guards to be placed inside a component that is a child of `SessionProvider` — the root layout itself must split into two: an outer `RootLayout` that wraps `<SessionProvider>` and an inner `RootNavigator` that reads `useSession()` and renders `Stack.Protected`. If you try to call `useSession()` in the same component that renders `<SessionProvider>`, the context won't be available. |
| K026 | M003/S05/T01 | expo | Expo SDK 54 `expo-file-system` v19 uses a completely new class-based API: `File`, `Directory`, `Paths` instead of the legacy `FileSystem.cacheDirectory`, `FileSystem.downloadAsync()`. Use `import { File as ExpoFile, Paths } from 'expo-file-system'` (aliased to avoid conflict with global `File`). Download: `ExpoFile.downloadFileAsync(url, new ExpoFile(Paths.cache, name), { headers })`. Legacy methods exist but throw at runtime with deprecation errors. |
| K027 | M003/S06/T02 | expo | Expo SDK 54 `expo-notifications` ~0.32.x changed `NotificationBehavior` — `shouldShowAlert` is deprecated, replaced by `shouldShowBanner` and `shouldShowList` (both required booleans). TypeScript compilation fails if you only provide `shouldShowAlert`. Always include all three for backward compatibility. |
